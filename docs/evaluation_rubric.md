# Evaluation Rubric



Each coding agent attempt is evaluated from 0 to 5.



## Categories



### 1. Correctness



Does the solution satisfy the task requirements?



### 2. Simplicity



Does the solution avoid unnecessary complexity?



### 3. Readability



Is the code easy to understand and maintain?



### 4. Error Handling



Does the solution handle invalid input or edge cases?



### 5. Testability



Can the behavior be verified through tests or clear manual steps?



### 6. Constraint Following



Did the agent follow all stated constraints?



### 7. Git Hygiene



Are commits, diffs, and explanations clean and reviewable?



## Score Meaning



- 5: Excellent

- 4: Good

- 3: Acceptable

- 2: Weak

- 1: Poor

- 0: Missing or harmful
